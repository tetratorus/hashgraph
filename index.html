Hashgraph Consensus

An introduction to hashgraph consensus by Stefan Buhrmester

What is Hashgraph Consensus?

First conceived by Leemon Baird, the hashgraph consensus algorithm is a generic algorithm that runs on connected nodes of a distributed system and maintains a consensus over the correct order of events that occur among the nodes that make up the system.

It is particularily useful for the operation of distributed systems that need to operate on a globally agreed state, such as log replication, distributed ledgers or shared databases.

It operates without leader election, is optionally byzantine, runs in a dynamic environment, and depending on implementation can be very bandwidth-efficient.
